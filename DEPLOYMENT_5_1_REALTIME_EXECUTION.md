## ⚠️ 遇到问题时的处理流程

### 如果部署卡在某个步骤

```
1. 记录问题描述 (2 分钟)
   - 错误信息完整记录
   - 当前系统状态截图
   - 已尝试的解决方案

2. 尝试快速恢复 (5 分钟)
   - 参考文档故障排查部分
   - 尝试基本解决步骤
   
3. 报告给运维 Lead (立即)
   - 描述问题和已尝试的解决方案
   - 提供日志和错误信息
   
4. 集体讨论 (5 分钟)
   - 运维 Lead + 开发主力 + 相关人员
   - 评估是否需要回滚
   
5. PM 决策
   - 继续推进 (如果是小问题)
   - 部分回滚后重试
   - 完全回滚并重新规划
```

### 严重问题的上报路线

```
级别 1 - 普通问题 (预计 5-10 分钟解决)
  报告给: 运维 Lead
  示例: Java 版本不对、Nginx 配置语法错误
  处理: 直接修复
  
级别 2 - 中等问题 (预计 15-30 分钟解决)
  报告给: 运维 Lead + 开发主力
  示例: 应用无法启动、数据库连接失败
  处理: 团队讨论后决策
  
级别 3 - 严重问题 (可能需要回滚)
  报告给: PM + 运维 Lead + 技术主管
  示例: 系统无法恢复、关键服务宕机
  处理: PM 宣布暂停，启动应急预案
```

## 🔄 回滚程序 (如需要)

**如果 5/1 部署失败，执行回滚**:

```bash
# 1. 停止应用
sudo systemctl stop campus-app.service

# 2. 恢复 Nginx 配置
sudo systemctl stop nginx
sudo cp /etc/nginx/conf.d/campus.conf /etc/nginx/conf.d/campus.conf.bak
# 手动编辑或从备份恢复原始配置
sudo systemctl start nginx

# 3. 清理已安装的新包 (如需要)
sudo yum remove -y java-17-openjdk java-17-openjdk-devel  # CentOS
# 或
sudo apt purge -y openjdk-17-jdk openjdk-17-jre            # Ubuntu

# 4. 恢复系统配置
sudo tar -xzf /tmp/backup-20260501.tar.gz -C /

# 5. 验证系统恢复
sudo systemctl status
curl http://localhost  # 检查原有服务
```

**回滚后流程**:
- [ ] PM 宣布 5/1 暂停，计划重试时间
- [ ] 团队召开问题分析会议 (30 分钟)
- [ ] 确认根本原因和解决方案
- [ ] 修改部署计划或脚本
- [ ] 宣布新的执行时间
