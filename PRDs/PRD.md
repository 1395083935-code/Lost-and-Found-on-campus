校园失物招领小程序完整版产品开发文档
文档版本与说明
1.1 文档信息
•版本号：V1.0
•适用场景：校园内部失物招领轻量化小程序，面向在校学生、教职工，配套管理员后台管理
•开发工具：适配Cursor编辑器开发，基于微信小程序原生语法/Uniapp开发，后端采用轻量化框架（如Node.js/Java SpringBoot）
•开发原则：优先实现核心闭环，兼顾用户体验、数据安全、隐私保护，支持后续功能迭代
1.2 产品核心目标
解决校园内物品遗失后找回渠道单一、信息传播范围小、线下招领处信息不透明的痛点，搭建线上发布-智能匹配-审核公示-沟通交接-完结归档的完整失物招领流程，提升物品找回效率，打造便捷、可信的校园便民工具。
1.3 用户角色界定

用户角色	权限与操作	使用场景
普通未登录用户	浏览招领信息、搜索信息、查看使用帮助	临时查找物品，未注册使用
普通登录用户	微信授权登录、失物/拾物发布、信息搜索、一键联系、个人中心管理、信息分享	学生/教职工丢失或拾获物品
系统管理员	后台登录、信息审核、信息管理、用户举报处理、数据统计、公告发布	校园后勤/学生会管理人员维护平台
二、产品整体架构设计
2.1 技术架构
•前端：微信小程序端（原生WXML+WXSS+JS/Uniapp）、管理员Web后台（Vue/ElementUI）
•后端：轻量化服务端接口，RESTful API规范
•数据库：MySQL（轻量版），存储用户信息、招领信息、审核记录、操作日志
•第三方服务：微信授权登录、微信分享、图片云存储、短信验证码（可选）
2.2 功能模块总览
1.微信小程序端：登录授权模块、首页模块、信息发布模块、信息详情模块、个人中心模块、消息通知模块、帮助中心模块
2.管理员后台端：登录认证模块、信息审核模块、信息管理模块、用户管理模块、数据统计模块、系统设置模块、公告管理模块
三、微信小程序端详细功能需求
3.1 登录授权模块
3.1.1 登录逻辑
•支持微信快捷授权登录（核心），可选校园学号/工号绑定登录（提升真实性），未登录用户仅拥有浏览权限，无发布、联系、分享权限
•登录流程：进入小程序→弹窗授权微信昵称、头像→获取微信openid唯一标识→注册/登录成功→跳转首页
•自动登录：首次登录后，下次进入小程序自动登录，无需重复授权
•退出登录：个人中心支持一键退出，退出后清空本地登录状态
3.1.2 隐私与权限
•仅获取微信公开信息（昵称、头像、openid），不获取手机号、位置等敏感信息（位置需用户手动授权）
•用户信息加密存储，禁止泄露给第三方
3.2 首页模块
3.2.1 页面布局
3.顶部区域：搜索框（支持关键词模糊搜索）、公告栏（滚动展示管理员发布的校园招领通知）
4.筛选区域：分类筛选（证件类、电子产品、文具饰品、衣物箱包、校园卡/证件、其他）、时间筛选（全部、1天内、3天内、7天内）、类型筛选（失物寻主、拾物招领）
5.内容区域：双标签页切换（失物寻主、拾物招领），瀑布流展示信息卡片
6.底部操作区：悬浮【发布】按钮、导航栏（首页、个人中心）
3.2.2 信息卡片展示
•卡片内容：物品缩略图（无图显示默认图标）、物品名称、丢失/拾获地点、发布时间、距离当前时间（如：2小时前）
•卡片状态：未完结信息正常展示，已完结信息标注【已找回/已归还】标签，不进入推荐列表
•加载逻辑：下拉刷新、上拉加载更多，无数据时显示空状态提示（暂无相关招领信息）
3.2.3 搜索功能
•支持物品名称、丢失/拾获地点、物品描述关键词模糊搜索
•搜索结果支持按时间、相关性排序，无结果时显示“未找到相关信息，建议发布寻物启事”
3.3 信息发布模块
3.3.1 发布入口
首页悬浮【发布】按钮，点击选择发布类型：我丢失了物品（失物发布）、我拾获了物品（拾物发布）
3.3.2 失物发布表单（必填+选填明确）

字段名称	字段类型	是否必填	规则说明
物品名称	输入框	是	限制2-20字，如：校园卡、黑色蓝牙耳机
丢失时间	日期时间选择器	是	不可选择未来时间，默认当前时间
丢失地点	输入框+快捷选择	是	快捷选项：教学楼、食堂、图书馆、操场、宿舍区、体育馆，支持自定义输入
物品分类	下拉选择框	是	固定分类：证件类、电子产品、文具饰品、衣物箱包、其他
物品特征描述	多行输入框	否	限制500字内，填写物品细节（如：校园卡印有XX学院，耳机有划痕）
物品照片	图片上传	否	支持上传1-3张，单张≤2M，自动压缩，支持预览、删除
联系方式	手机号输入框	是	验证手机号格式，支持脱敏显示
匿名发布	开关按钮	否	开启后隐藏微信昵称、头像，仅显示联系方式
提交按钮	按钮	-	点击校验必填项，校验通过提交，显示“提交成功，等待管理员审核”
3.3.3 拾物发布表单（必填+选填明确）

字段名称	字段类型	是否必填	规则说明
物品名称	输入框	是	限制2-20字
拾获时间	日期时间选择器	是	不可选择未来时间，默认当前时间
拾获地点	输入框+快捷选择	是	同失物发布快捷选项
物品分类	下拉选择框	是	同失物分类
物品特征描述	多行输入框	否	限制500字内
物品照片	图片上传	否	支持1-3张，自动压缩
存放地点	输入框	是	填写：本人宿舍、校园失物招领处、XX办公室等
联系方式	手机号输入框	是	验证手机号格式
匿名发布	开关按钮	否	开启后隐藏个人信息
提交按钮	按钮	-	校验后提交，等待审核
3.3.4 发布限制
•同一用户每日最多发布5条信息，防止刷屏
•重复信息（相同物品、地点、时间）系统自动提示，禁止提交
•提交后不可修改，仅可在个人中心编辑后重新提交
3.4 信息详情模块
7.详情展示：完整展示发布表单所有信息，包括照片、时间、地点、描述、联系方式（手机号中间四位脱敏）
8.操作按钮：
◦一键拨号：调用系统拨号功能，拨打脱敏后手机号，不直接显示完整号码
◦分享好友：分享单条信息至微信好友、微信群
◦收藏信息：登录用户可收藏，方便后续查看
◦举报信息：针对虚假/违规信息，选择举报类型（虚假信息、垃圾广告、恶意骚扰），提交举报说明
9.状态标识：审核中、待认领、已找回、已归还、已驳回
10.返回逻辑：点击返回按钮，回到上一级页面（首页/搜索结果页/个人中心）
3.5 个人中心模块
3.5.1 基础信息展示
•展示微信头像、昵称，学号/工号（绑定后显示）
•展示诚信标识（累计归还物品数量，无负面举报）
3.5.2 我的发布管理
•分类标签：待审核、已通过、已驳回、已完结
•操作权限：
◦待审核：编辑信息、撤销发布、查看审核进度
◦已通过：删除信息、标记【已找回/已归还】、编辑重新提交
◦已驳回：查看驳回原因、重新编辑提交、删除
◦已完结：仅查看、删除记录
•状态同步：标记完结后，信息自动从首页下架，移入已完结列表
3.5.3 我的收藏
展示收藏的所有招领信息，支持取消收藏、一键拨号、分享
3.5.4 消息通知
展示审核结果通知、匹配提醒、举报处理结果通知，已读/未读标识，支持批量删除
3.5.5 其他功能
•使用帮助：发布流程、审核规则、交接须知、常见问题
•意见反馈：输入反馈内容+上传图片，提交给管理员
•关于我们：小程序介绍、开发团队、联系客服
3.6 辅助功能模块
11.智能匹配提醒：系统根据用户发布的失物/拾物信息，自动匹配相似信息，推送匹配通知
12.信息时效：已通过信息默认展示30天，到期自动隐藏，用户可重新发布
13.深色模式：适配微信深色/浅色模式，自动切换样式
14.离线缓存：缓存近期浏览信息，无网络时可查看基础内容
四、管理员后台详细功能需求
4.1 登录认证模块
•账号密码登录，支持验证码验证，防止暴力破解
•单账号登录限制，同一账号仅可在一处登录
•密码修改、退出登录功能
4.2 信息审核模块
15.待审核列表：展示所有用户提交的失物/拾物信息，按提交时间倒序排列
16.审核操作：
◦通过：信息同步至小程序首页公开
◦驳回：填写驳回原因（虚假信息、信息不全、违规内容、重复信息），通知用户
17.批量审核：支持多选信息，批量通过/驳回
18.信息预览：点击查看信息完整详情，包括照片、描述、联系方式
4.3 信息管理模块
19.全部信息列表：展示所有审核通过、已驳回、已完结信息，支持搜索、分类、时间筛选
20.信息操作：编辑、删除、置顶、标记完结、导出信息报表
21.举报处理：查看用户举报信息，核实后处理（删除信息、警告用户、封禁账号）
4.4 用户管理模块
22.用户列表：展示所有登录用户信息，搜索、筛选用户
23.用户操作：查看用户发布记录、举报记录，封禁违规用户（禁止发布/登录）、解除封禁
24.用户数据：统计用户发布量、找回率、诚信等级
4.5 数据统计模块
25.核心数据：今日/本周/本月发布量、审核量、完结量、找回率、用户活跃度
26.数据可视化：折线图展示发布趋势、饼图展示物品分类占比、高频遗失/拾获地点统计
27.数据导出：导出Excel报表，支持按时间、分类筛选导出
4.6 系统设置模块
28.公告管理：发布、编辑、删除小程序首页滚动公告，设置公告生效时间
29.分类管理：添加、修改、删除物品分类
30.审核规则设置：设置自动审核关键词、发布限制数量、信息时效
31.反馈管理：查看用户反馈，回复处理结果

六、接口规范（Cursor开发参考）
32.接口格式：统一RESTful API，请求方式GET/POST，返回格式JSON
33.返回参数：code（状态码）、msg（提示信息）、data（数据内容）
◦code=200：请求成功
◦code=400：参数错误
◦code=401：未登录/权限不足
◦code=500：服务器错误
34.接口加密：敏感接口（登录、发布）采用token验证，token有效期2小时
35.图片上传：上传至云存储，返回图片URL，限制单张2M以内
七、交互与视觉规范
36.视觉风格：简洁清新，校园风，主色调采用蓝色/绿色，符合校园工具定位
37.交互逻辑：操作流程极简，表单填写步骤少，按钮点击有反馈，加载状态有提示
38.适配要求：适配所有手机尺寸，无样式错乱，支持微信基础以上版本
八、开发与测试要求
8.1 开发要求
•代码规范：注释清晰，变量命名规范，便于Cursor编辑器代码提示与补全
•模块化开发：按功能模块拆分代码，降低耦合度，方便后续迭代
•性能优化：图片压缩、懒加载，减少接口请求次数，提升页面加载速度
8.2 测试要点
39.功能测试：所有功能正常使用，表单校验、审核流程、状态同步无误
40.兼容性测试：安卓、iOS不同机型、微信版本适配正常
41.安全测试：用户信息加密，防SQL注入，防恶意发布
42.性能测试：页面加载≤2秒，接口响应≤1秒
