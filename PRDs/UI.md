.Cursor 开发建议：前端框架选择
根据文档中 1.1 开发工具 和 2.1 技术架构 的描述，文档明确指出了“基于微信小程序原生语法/Uniapp开发”。
为了在 Cursor 中获得最佳的 AI 代码补全（Code Completion）和开发体验，建议如下：
核心推荐：UniApp (Vue 3 + Vite)
理由：
文档指定：你的产品文档明确提到了 Uniapp，这意味着项目需要跨端（小程序+可能的H5/APP）或者团队熟悉 Vue 生态。
Cursor 友好：Cursor 对 Vue 3 的 <script setup> 语法支持极好，AI 能更准确地理解组件结构。
组件库丰富：可以使用 uView 或 ThorUI 等成熟组件库，快速搭建 UI。
Cursor 初始化配置建议：
项目模板：在 Cursor 中选择 Vue 3 模板，然后安装 UniApp CLI 或直接使用 HBuilderX 标准模板（将代码放入 Cursor）。
目录结构：
src/
├── pages/               # 页面文件
│   ├── index/           # 首页
│   │   ├── index.vue
│   ├── publish/         # 发布页
│   │   ├── publish.vue
│   └── detail/          # 详情页
│       ├── detail.vue
├── components/          # 公共组件 (如: CardItem, FilterBar)
├── utils/               # 工具类 (如: request.js 封装)
├── api/                 # 接口定义 (对接文档 6. 接口规范)
└── uni.scss             # 全局样式
备选方案：微信小程序原生 (React 语法系)
理由：
如果团队追求极致的性能且只做微信小程序，文档中也提到了“原生WXML”。
Cursor 配置：选择 JavaScript 或 TypeScript 环境。
注意：原生开发在 Cursor 中的 AI 补全体验不如 Vue/React 框架流畅，需要更多手动编写 WXML 结构。
组件库推荐 (用于 Cursor 开发)
为了快速实现上面的 UI 草图，建议在 Cursor 中引入以下库：
uView UI (UniApp 最佳搭档，文档丰富，符合文档中“轻量化”要求)。
Pinia：用于状态管理（管理用户登录态 token 和 userInfo）
